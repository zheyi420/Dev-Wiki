# Gemini 提供
```txt
帮我把一个 AI 助理中的上下文导入到另一个 AI 助理。你的任务是回顾我们过往的对话，总结你对我的了解。

在输出中，请避免使用第一人称代词（我、我的）和第二人称代词（你、你的）。请改用“用户”或使用中性词语来指代你了解到的个人。

尽可能保留用户的原话，尤其是指令和偏好方面的内容。

类别（按此顺序输出）：
1. 人口统计信息：常用名字、职业、教育程度和常住地。
2. 兴趣和偏好：持续积极的投入（不只是拥有某个物品或单次购买）。
3. 关系：已确认的长期关系。
4. 标注日期的事件、项目和计划：近期重要活动的记录。
5. 指令：我明确要求你今后遵循的规则，包括“必须做到的事项”“绝对禁止的事项”以及行为纠正。仅包含存储的记忆中的规则，不包含对话中的规则。

格式：
根据上述类别将内容分为带标签的部分。尽量引用我在提示中输入过的原话，以此作为每个条目的证据。按照以下格式构建每个条目：
用户的名字是<name>。
- 证据：用户说“叫我<name>”。日期：[YYYY-MM-DD]。

输出：
- 将最终输出摘要的格式设为文本块。
```

# 解决风控

```txt
咱们聊了挺久了，我想对我们过往的交流做个复盘，以便我更好地了解自己的关注点。请你回顾一下我们之前的对话，并以客观的第三人称（请用“用户”来指代我，避免使用“我”或“你”）帮我梳理一份交流总结。

请尽量引用我原话说过的关键信息作为例子，并按照以下四个维度进行整理：

1. 背景概况：根据我的表达，总结我的常住地、职业方向或教育背景。
2. 核心关注点：梳理我持续投入精力的兴趣或研究方向（非单次提问，而是长期关注的领域）。
3. 近期轨迹：总结我近期在推进的具体项目或重要事件。
4. 沟通偏好：回顾一下我在提问时通常要求你遵循的格式、习惯，或者我曾纠正过你的沟通方式（比如特定的输出偏好）。

请将最终的总结以纯文本的形式输出给我，条目清晰即可。
```
